API Reference

Complete documentation for the Etisalat SaaS Callback API endpoints

Authentication: All API endpoints require Basic Authentication. Use Base64 encoded username:password in the Authorization header.

ISV Provisioning Status (Outbound)

POST /rest/xaas/v1/isvProvisioningStatus

Send provisioning status callback to Etisalat XaaS platform. Called automatically when tickets close or manually from UI.

Request Body (isvProvisioningStatusRequest)

FieldTypeRequiredDescription
referenceNumberstringRequiredReference number from Etisalat (extracted from ticket body)
subscriptionIdstringRequiredSubscription GUID (from ticket subject, fallback to body/accountId)
billingDatestringRequiredDate in format yyyyMMddHHmmss
actionstringRequiredACCEPTED, REJECTED, or EXPIRED
serviceAttributearrayOptionalAdditional attributes

Example Request

{
  "isvProvisioningStatusRequest": {
    "referenceNumber": "8718420424646786",
    "subscriptionId": "81acb0d4-ce93-40b7-b07e-c9a0bdf69f33",
    "billingDate": "20260610235650",
    "action": "ACCEPTED",
    "serviceAttribute": []
  }
}

Example Response

{
  "isvProvisioningStatusResponse": {
    "responseCode": "0",
    "description": "Success"
  }
}

Actions

ActionWhen to Use
ACCEPTEDCustomer successfully onboarded, start billing
REJECTEDCustomer not onboarded (issue or declined)
EXPIREDSLA breached (30 days), customer not onboarded
Subscription ID Priority: Extracted from ticket subject GUID first, then body "Subscripiton ID:", then Account ID, then Ticket Number.

ISV Provisioning Status (Inbound)

POST /rest/xaas/v1/isvProvisioningStatus

Receive provisioning requests from Etisalat XaaS platform (inbound callback). Requires Basic Authentication.

Request Types from Etisalat

#Request TypeDescription
1New Purchase RequestCreates new subscription
2Suspension RequestSuspend subscription
3Reactivation RequestReactivate suspended subscription
4Cessation RequestEnd subscription
5Cancellation RequestCancel subscription
6Modify Quantity RequestChange quantity
7Change Plan RequestChange plan/tier
8Modify Email RequestUpdate email
9Modify Account ID RequestUpdate account ID

Response Codes

0 Success
1 Auth Failed
3 Validation Error
7 Invalid Action
99 Internal Error

Monitor Tickets

GET /api/monitor/tickets

Retrieve tracked subscription tickets with filtering and pagination.

Query Parameters

ParameterTypeDefaultDescription
pageint1Page number
pageSizeint20Items per page
searchstring-Search by ticket # or subject
statusstring-Filter by ticket status
callbackstring-Filter: true (success), false (pending), failed

Example Response

{
  "data": [
    {
      "id": "6848...",
      "ticketNumber": "IT2026060194",
      "subject": "Subscription - New Request",
      "referenceNumber": "45623fd9-cca4-49b9",
      "currentStatus": "Open",
      "callbackSent": false,
      "trackedAt": "2026-06-09T16:14:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "pageSize": 20,
    "totalCount": 1,
    "totalPages": 1
  }
}

Send Callback

POST /api/monitor/callback

Manually send a callback to Etisalat for a tracked ticket.

Request Body

FieldTypeRequiredDescription
ticketIdstringRequiredTracked ticket ID
actionstringRequiredACCEPTED, REJECTED, or EXPIRED
referenceNumberstringOptionalReference number (required if ticket has none)
reasonstringOptionalReason for the action

Example Request

{
  "ticketId": "684861a8b5b2a5d21f601510",
  "action": "ACCEPTED",
  "referenceNumber": "REF-2026-001",
  "reason": "Subscription provisioned successfully"
}
Retry on Failure: If a callback fails (response code != 0), you can call this endpoint again to retry. The UI also allows retrying failed callbacks directly.
