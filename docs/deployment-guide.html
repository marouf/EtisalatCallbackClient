Deployment Guide

Step-by-step instructions for deploying the Etisalat SaaS Callback Service

Prerequisites

ComponentVersionPurpose
.NET SDK10.0+Application runtime
MongoDB8.0+Database
Docker24.0+Containerization (optional)
GitAnySource control

Deployment Options

Recommended: Easiest method - includes MongoDB automatically
1
Navigate to project directory 
cd /path/to/EtisalatSaasCallback
2
Build and start services 
# Build and run in detached mode
docker-compose up -d --build

# View logs
docker-compose logs -f api

# Check status
docker-compose ps
3
Verify deployment 
# Test health endpoint
curl http://localhost:5000/api/health

# Access services
# API:          http://localhost:5000
# Dashboard:    http://localhost:5000/Dashboard
# Swagger:      http://localhost:5000/swagger
# Mongo Express: http://localhost:8081
1
Install .NET 10 Runtime 
dotnet --version
2
Restore and build 
dotnet restore
dotnet build -c Release
dotnet publish -c Release -o ./publish
3
Run the application 
cd publish
set ASPNETCORE_ENVIRONMENT=Production
dotnet EtisalatSaasCallback.dll --urls "http://0.0.0.0:5000"
1
Install .NET Runtime 
sudo apt-get update
sudo apt-get install -y aspnetcore-runtime-10.0
2
Create systemd service 
sudo tee /etc/systemd/system/etisalat-saas.service << 'EOF'
[Unit]
Description=Etisalat SaaS Callback Service
After=network.target mongodb.service

[Service]
WorkingDirectory=/var/www/etisalat-saas
ExecStart=/usr/bin/dotnet EtisalatSaasCallback.dll
Restart=always
Environment=ASPNETCORE_ENVIRONMENT=Production
Environment=ASPNETCORE_URLS=http://localhost:5000

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl enable etisalat-saas
sudo systemctl start etisalat-saas
3
Configure Nginx reverse proxy 
sudo apt-get install -y nginx

sudo tee /etc/nginx/sites-available/etisalat-saas << 'EOF'
server {
    listen 80;
    server_name api.yourdomain.com;

    location / {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
EOF

sudo ln -s /etc/nginx/sites-available/etisalat-saas /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx

Post-Deployment Checklist

  • Application starts without errors
  • Health endpoint returns "healthy"
  • MongoDB connection working
  • HTTPS enabled and working
  • Etisalat credentials configured
  • IP whitelisting enabled for production
  • Dashboard login working (default admin from UiAuth settings)
  • Default admin password changed
  • User management accessible (/Users - Admin only)
  • Ticket monitoring service running
  • Search by ticket # and subject working
  • Failed callback filter and retry working
  • Logs directory writable
